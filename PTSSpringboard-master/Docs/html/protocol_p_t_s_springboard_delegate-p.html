Detailed Description

The protocol is implemented by a PTSSpringboard delegate to be informed of user-interactions occuring with the springboard. All the methods in this protocol are optional.

- (void) springboard: (PTSSpringBoard *)  springboard
deletedItem: (PTSSpringboardItem *)  item
atIndex: (NSInteger)  index 
optional

The PTSSpringboard informs the delegate that an item was deleted. This method is only called, while the springboard is in editing mode. You should use it to update your data source. If you don't the springboard will return to its original state, as soon as so leave editing mode.

Parameters
springboardA reference to the calling PTSSpringboard.
itemA reference to the PTSSpringboardItem that was tapped.
indexThe positional index of the item, that was taped.

- (void) springboardDidEnterEditingMode: (PTSSpringBoard *)  springboard
optional

The PTSSpringboard informs the delegate, that it has entered editing mode. This triggered by calling the PTSSpringboards -(void)toggleEditingMode: method. By default, a long press on one of the items will call this method.

Parameters
springboardA reference to the calling PTSSpringboard.
- (void) springboardDidLeaveEditingMode: (PTSSpringBoard *)  springboard
optional

The PTSSpringboard informs the delegate, that it has left editing mode. By default this triggered by calling the PTSSpringboards. It is up to the controller to make a PTSSpringboard leave editing mode.

Parameters
springboardA reference to the calling PTSSpringboard.
